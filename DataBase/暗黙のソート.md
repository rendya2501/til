# 暗黙のソート

---

## 概要

Group By や Distinct によりデータ抽出する場合に Order By を指定しなくてもキー項目順でソートされる機能のこと。  

---

## 色々

>暗黙のソートが実行されるかどうかはデータベースによる。  
基本的に「order by区なしでは順序保証されない」と明記されていることが多い模様。  
SQL Server では「primary key」指定時に、「order by」なしでも順序保証しているようです。  
ですが、基本的にOrderBy区を省略してSQL文を発行してはいけないと思います。  
他の人のプログラムを修正したときのトラップのうちの一つですね。  
[暗黙ソートについて | SQL](https://itblogdsi.blog.fc2.com/blog-entry-7.html)

---

暗黙のソート順がどんなものになるのか調べるために1つの記事としてみたが、そもそも暗黙のソートで困ることがあるなら、素直にORDER BYすればいいだけの話だった。  
わざわざデータベースの仕様まで調べる必要はなかった。  
時間があるならそれでいいが、そこまでやる必要はない。  

結果を出力する以上、何かの仕様に沿ってソート処理はするだろう。  
必要ならORDER BYしろ。以上。
